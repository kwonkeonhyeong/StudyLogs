## 배열

배열은 어떻게 index를 이용해서 한번에 데이터를 찾을 수 있을까? 

간단 공식

x100(배열 시작 주소) + (자료의 크기 * index) 

위 공식을 이용하여 정확한 데이터 주소를 한번에 찾을 수 있음

배열의 순차 검색은 배열에 들어있는 데이터의 크기만큼 연산이 필요하다.

배열의 크기가 n이면 연산도 n만큼 필요하다!

- 배열의 인덱스 사용 : O(1)
- 배열의 순차 검색 : O(n)


## 배열의 데이터 추가

배열의 데이터를 중간에 추가하기 위해서는 기존의 데이터가 오른쪽으로 한 칸씩 이동해야 한다.

쉽게 말해 새로운 데이터가 들어가기 위한 공간을 확보해야 한다.

따라서 기존 데이터를 오른쪽으로 한 칸씩 밀어야 한다. (기존 데이터의 인덱스를 하나씩 증가시켜야 한다.)

배열의 데이터를 추가하는 위치에 따라 크게 3가지로 나눌 수 있다.

- 배열의 첫번째 위치에 추가
- 배열의 중간 위치에 추가
- 배열의 마지막 위치에 추가

## 배열의 한계

배열은 기본적인 자료구조이고 인덱스를 사용할 때 최고의 효율이 나온다. 하지만 이런 배열에는 큰 단점이 존재한다.
바로 크기가 고정되어 있다는 점이다. 배열의 크기는 배열을 생성하는 시점에 미리 정해야 한다. 
배열의 크기를 미리 고정하게 되면 배열에 많은 데이터가 저장되었을 때 배열의 공간이 부족해지거나,
또는 처음부터 너무 많은 공간을 할당하여 공간을 낭비하게 될 수 있다.

배열처럼 처음부터 정적으로 길이가 정해져있는 것이 아니라, 동적으로 언제든지 길이를 늘이고 줄일 수 있는 자료 구조가 있다면 편리할 것이다.
