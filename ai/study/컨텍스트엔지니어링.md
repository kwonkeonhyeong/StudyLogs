## 페목형제

1. 페르소나 (Persona)

누구의 관점에서 말할 것인가?? 

페르소나를 지정하여 AI가 탐색해야 할 정답의 범위를 좁혀준다.

2. 목표 (objective)

"알아서 잘", "적당히" 같은 모호한 표현은 사용하지 않는다.

원하는 결과물이 무엇인지 구체적으로 정의한다.

3. 형태 (format)

어떤 형태를 원하는가? 처음부터 원하는 결과물 형식을 명확히 지정하여, AI 결과물이 곧바로 다음 공정에 투입될 수 있는 완제품이 되도록 한다.

4. 제약 조건 (Constraints)

무엇을 '하지 말아야' 하는가?

제약조건은 AI가 헛다리를 짚거나 잘못된 길로 빠지지 않도록 안전 울타리를 쳐주는 역할을 한다.

## 프롬프트 디자인 패턴

1. 템플릿 제공 패턴 (Few-shot Prompting)

AI에게 실제 과업을 지시하기 전에, 우리가 원하는 결과물의 형식과 스타일을 보여주는 방법

**언제 사용해야 할까요?**

- 결과물의 **일관된 톤앤매너나 스타일**이 매우 중요할 때 (e.g., 블로그 글, SNS 카피라이팅)
- 정해진 데이터 형식(JSON, XML 등)으로의 변환이 필요할 때
- **복잡한 규칙 기반의 텍스트 생성**이 필요할 때


2. 단계적 분해 패턴 (Step-by-step Decomposition, Chain of Thought)

- LLM의 가장 큰 약점 중 하나는 복잡한 문제 앞에서 성급하게 결론으로 건너뛰려는 경향이다.
- AI는 자신의 추론 과정을 텍스트로 풀어쓰는 과정에서 논리적 오류를 스스로 발견하고 수정하게 된다.
