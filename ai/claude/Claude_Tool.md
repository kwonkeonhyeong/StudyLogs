클로드 코드를 만든 Boris Cherny가 처음 가졌던 아이디어는 AI에게 도구를 자유롭게 쓰게 해주면 어떻게될까였다고 이야기한 적이 있다. 에이전틱 코딩의 핵심은 AI 모델이 사용자의 환경에서 활용 가능한 도구를 이용하면서 문제를 풀어가게 하는 것이다.

그렇다면 이런 AI 코딩 도구를 사용해서 일을 할 때 어떤 도구들이 제공되고, 그게 어떻게 활용되는지를 먼저 살펴보는 것이 필요하다.

그래서 나는 종종 AI 도구에게 현재 가지고 있는 도구가 뭔지를 물어본다.
예를 들어 클로드 코드에게 물어보면, 우선 기본 사용 도구를 알려준다.

-------------------
⏺ 제가 사용할 수 있는 도구들은 다음과 같습니다:

파일 작업

- Read - 파일 내용 읽기 (이미지, PDF, Jupyter 노트북 포함)
- Write - 새 파일 작성 또는 기존 파일 덮어쓰기
- Edit - 파일 내 특정 문자열 교체
- Glob - 패턴으로 파일 검색 (예: **/*.ts)
- Grep - 파일 내용에서 정규식 패턴 검색
- NotebookEdit - Jupyter 노트북 셀 편집

명령 실행

- Bash - 터미널 명령어 실행 (git, npm, docker 등)
- KillShell - 백그라운드 쉘 프로세스 종료

에이전트 & 태스크

- Task - 복잡한 작업을 위한 전문 에이전트 실행
- Bash - 명령어 실행 전문
- Explore - 코드베이스 탐색 전문
- Plan - 구현 계획 설계
- general-purpose - 범용 연구/검색
- TaskOutput - 백그라운드 태스크 출력 확인

웹 & 검색

- WebFetch - URL에서 콘텐츠 가져와 분석
- WebSearch - 웹 검색 수행

계획 & 관리

- TodoWrite - 작업 목록 생성 및 관리
- EnterPlanMode - 구현 계획 모드 진입
- ExitPlanMode - 계획 모드 종료

사용자 상호작용

- AskUserQuestion - 사용자에게 질문하여 명확화
- Skill - 사용자 정의 스킬 실행 (예: /commit, /code-explore)

총 17개의 도구를 활용하여 소프트웨어 엔지니어링 작업을 도와드릴 수 있습니다.
-------------------

클로드 코드에서 작업을 진행할 때 이 이름의 도구들이 사용되는 것을 볼 수 있다. 나는 이걸 좀 더 명확하게 확인하기 위해서 클로드 코드가 도구를 사용하기로 결정했을 때 앞에 당근을 붙여달라고 했다. 그러면,

- 🥕 Task (Explore 에이전트) 사용하여 프로젝트를 분석합니다.
- Explore(Analyze project structure)

- 🥕 Glob 사용하여 YML 파일을 검색합니다.
- Search(pattern: "**/*.yml")

이런식으로 알려준다.

그리고 각 Tool 들에 대해서 자세히 설명하고, 활용방법을 알려달라고 한다. 그러면 해당 도구의 완벽 가이드를 만들어준다. 도구의 특성을 파악하면 이를 잘 활용할 수 있는 방법으로 요구를 다듬을 수 있다.

코드를 분석할 때는 Explore 서브 에이전트를 사용하는 것이 가장 좋다. 이건 병렬로 수행도 가능하다. 그러면 구체적으로 Explore 서브 에이전트를 병렬로 사용해서 모듈별로 분석을 해달라고 요청을 할 수 있다. 그냥 코드를 분석해달라고 할 때와 동작 방식이 달라진다. 또, 내 요구사항에 명확하게 확인하고 진행 하도록 AskUserQuestion을 사용해달라고 할 수도 있다.

그리고 클로드 코드에는 기본 내장 sub-agents가 있다. 이것도 알아두는 게 좋다. 이것도 직접 물어보면 된다.

---

물론 현재 사용 가능한 스킬도 알려준다. 내가 추가한 커스톰 커맨드나 스킬 등도.

AI 에이전틱 코딩 도구로 개발을 잘 하려면 활용 가능한 도구를 파악하고, 이를 이용하는 능력이 요구된다. 필요하면 도구를 개발자가 직접 정의하고 이를 쓰도록 요구할 수 있다.

클로드 코드가 클로드에 보내는 request를 살펴보면, 사용자 프롬프트와 함께 클로드 코드가 사용가능한 툴 목록이 전달된다. 어떤 걸 사용할지 AI 모델에게 알려주고, 그걸 이용해서 작업을 어떻게 수행할지를 전달 받는다. 이게 에이전틱 코딩 도구가 동작하는 기본 방식이다.

도구는 버전업 되면서 계속 추가되기도 한다. 세세하게 분석해보면 흥미로운 게 많다. 예를 들어 Bash는 도구이기도 하고, sub-agent이기도 하다. 두 가지가 어떤식으로 다르게 활용되는지 AI한테 물어보고 아이디어를 얻을 수 있다.

클로드 코드 내장 도구나 서브 에이전트가 어떻게 작성된 코드인지는 알지 못하지만, 어떤 파라미터로 정의됐는지는 API 메시지를 분석해보면 알 수 있다. 그것도 활용에 도움이 된다.

커스톰 도구, 커맨드, 에이전트, 스킬까지 활용 가능한 방법이 많은데 이를 개발자나 팀이 다듬어가면서 개발을 하다보면 도구와 코드를 함께 만들어가는 느낌이 들 것이다. 이게 AI 에이전틱 코딩의 재미가 아닐까 싶다.

사용 예시

Task(subagent_type="Explore", prompt="API 엔드포인트 구조 파악해줘", thoroughness="medium")

# 구현 계획 수립
Task(subagent_type="Plan", prompt="사용자 인증 기능 구현 계획 세워줘")

# Git 작업
Task(subagent_type="Bash", prompt="최근 커밋 히스토리 분석해줘")

특징

- 여러 에이전트를 병렬로 실행 가능 (독립적인 작업일 경우)

- 이전 에이전트 재개 가능 (resume 파라미터)
